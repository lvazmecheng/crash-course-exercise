# Chapter 5: If Statements & Conditional Logic

## Overview
This chapter focuses on mastering flow control in Python. In the context of DevSecOps, `if` statements serve as the "Logic Gates" for automation determining how a script reacts to system states, user permissions, or environmental variables.

## Skills Demonstrated
* **Conditional Testing:** Evaluating equality, inequality, and multiple conditions using `and` / `or`.
* **List Logic:** Using the `in` and `not in` keywords to check for membership within a dataset.
* **Complex Branching:** Implementing `if-elif-else` chains to handle multiple distinct scenarios.
* **Data Validation:** Checking if lists are empty before processing to prevent runtime errors.

## Key Projects & Exercises

### 1. Username Validation (`checking_usernames.py`)
A simulation of a website registration system. 
* **Focus:** Case-insensitive comparison and list "hoisting" (moving list comprehensions outside loops) for optimized performance.

### 2. Ordinal Numbers (`ordinal_numbers.py`)
A script that converts integers (1–9) into their grammatical ordinal forms (1st, 2nd, etc.).
* **Focus:** Efficient use of `if-elif-else` blocks and string formatting with f-strings.

### 3. Future Project Roadmap (`Your_ideas.md`)
A strategic document outlining the transition from basic Python to complex engineering and web applications.
* **Projects Planned:** Restaurant Web Framework, API Integration, and a Graphical FEA Analysis tool.
